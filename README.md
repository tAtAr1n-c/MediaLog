# MediaLog

MediaLog - сайт для ведения личной медиатеки. Пользователь может искать реальные фильмы и книги, сохранять их, писать рецензии, смотреть отзывы других людей и получать случайные рекомендации.

Проект разделен на две независимые части:

- `frontend/` - клиентская часть на React;
- `backend/` - серверная часть на FastAPI.

Такую структуру удобно загружать на GitHub, показывать на защите и отдельно разворачивать frontend и backend на сервере.

## Основные возможности

- регистрация и вход пользователя;
- JWT-авторизация;
- личный профиль;
- сохранение фильмов и книг в библиотеку;
- создание и удаление рецензий;
- выбор фильма или книги для рецензии через поиск по реальным каталогам;
- отображение постеров фильмов и обложек книг;
- поиск других пользователей по нику или имени;
- просмотр открытых рецензий других пользователей;
- раздел "Лучшее за неделю";
- случайный фильм и случайная книга без обязательной регистрации;
- сохранение случайного фильма или книги после входа;
- страница поддержки.

## Запуск backend

Перейти в папку backend:

```bash
cd backend
```

Установить зависимости:

```bash
python3 -m pip install -r requirements.txt
```

Запустить сервер:

```bash
python3 -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Backend будет доступен по адресу:

```text
http://127.0.0.1:8000
```

## Запуск frontend

Открыть второй терминал и перейти в папку frontend:

```bash
cd frontend
```

Установить зависимости:

```bash
npm install
```

Запустить сайт:

```bash
npm run dev
```

Frontend будет доступен по адресу:

```text
http://127.0.0.1:5173
```

В режиме разработки Vite проксирует запросы `/api` на backend:

```text
http://127.0.0.1:8000
```

## Переменные окружения backend

Пример находится в файле:

```text
backend/.env.example
```

Можно создать рядом файл `backend/.env`:

```env
OMDB_API_KEY=your_omdb_key
GOOGLE_BOOKS_API_KEY=your_google_books_key
SECRET_KEY=your_secret_key
DATABASE_URL=sqlite+aiosqlite:///./app/review.db
FRONTEND_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

API-ключи не обязательны. Если ключей нет, backend использует открытые запасные источники.

## Как frontend связан с backend

Frontend отправляет запросы через файл:

```text
frontend/src/services/apiClient.js
```

По умолчанию API-адрес:

```text
/api
```

Во время локальной разработки Vite перенаправляет `/api` на FastAPI. На сервере такую же роль обычно выполняет Nginx.

## Как работает авторизация

1. Пользователь регистрируется через `/users/register`.
2. Для входа frontend отправляет логин и пароль на `/auth/token`.
3. Backend проверяет пароль и возвращает JWT-токен.
4. Frontend сохраняет токен в localStorage.
5. Для защищенных действий frontend отправляет заголовок:

```http
Authorization: Bearer <token>
```

6. Backend по токену определяет текущего пользователя.

## Как работают рецензии

Пользователь не вводит фильм или книгу как обычный текст. Сначала он ищет произведение через каталог, выбирает нужный вариант, а потом пишет рецензию.

В рецензию сохраняются:

- внешний ID;
- тип контента;
- название;
- автор, режиссер или тип;
- год;
- оценка;
- текст;
- ссылка на постер или обложку.

Данные хранятся в таблице `reviews`.

## Как работают сохраненные элементы

Сохраненные фильмы и книги хранятся отдельно от рецензий в таблице `user_contents`.

Один и тот же элемент нельзя сохранить дважды одному пользователю. Проверка идет по связке:

```text
user_id + external_id + media_type
```

## Как работает "Лучшее за неделю"

Backend берет рецензии за последние 7 дней, сортирует их по оценке и дате создания, а frontend показывает результат карточками.

## Как работают случайный фильм и случайная книга

Случайный фильм:

1. Backend берет список популярных поисковых запросов.
2. Перемешивает запросы.
3. Ищет фильмы во внешних источниках.
4. Сначала выбирает варианты с постером.
5. Возвращает случайный фильм из найденного списка.

Случайная книга:

1. Backend берет список тематических запросов.
2. Перемешивает запросы.
3. Делает запрос к источнику книг со случайным смещением.
4. Возвращает случайную книгу из найденных результатов.

Если пользователь авторизован, он может сразу сохранить случайный фильм или книгу.

## База данных

По умолчанию используется SQLite:

```text
backend/app/review.db
```

Основные таблицы:

- `users` - пользователи;
- `reviews` - рецензии;
- `user_contents` - сохраненные фильмы и книги.

При запуске backend автоматически создает таблицы, если их нет. Также он добавляет недостающие SQLite-колонки для старых версий базы.
